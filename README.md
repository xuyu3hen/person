# 迷亭桑的梦想生活 (Personal Digital Garden)

这是一个兼具静态前端展示与 Serverless 后台管理的个人数字花园项目。

## 项目简介

**项目背景与定位**：  
本项目旨在为科研人员或开发者提供一个一站式的数字展示与管理平台。它不仅是一个支持移动端适配、深浅色主题切换的个人主页（用于展示履历、开源项目和学术论文），更内置了一个高隐私的后台管理面板（Dashboard），用于记录私人日记、笔记、规划日常任务，以及本地化管理体型追踪。

**技术栈**：
- **前端框架**：`Next.js 15` (App Router) + `React 19`
- **样式库**：`Tailwind CSS v4` + `Lucide-React` 图标
- **数据库**：`PostgreSQL`（通过 `Neon Serverless` 连接）用于结构化数据；`IndexedDB` (`idb`) 用于客户端高隐私数据
- **组件与中间件**：`@dnd-kit`（拖拽排序）、`recharts`（数据可视化）、`@vercel/blob`（云端文件存储）
- **部署方式**：全静态构建或通过 `Vercel` / `Docker` 进行 Serverless 部署

**运行环境要求**：
- `Node.js` >= 20.0
- 包管理器：`npm` / `yarn` / `pnpm`

**快速启动步骤**：
1. 克隆代码后安装依赖：
   ```bash
   npm install
   ```
2. 复制环境变量模板并填入配置：
   ```bash
   cp .env.example .env.local
   ```
   *注意：你必须配置 `DATABASE_URL`（PostgreSQL 连接串）、`ADMIN_PASSWORD`（后台登录密码）以及 `BLOB_READ_WRITE_TOKEN`（Vercel Blob 存储凭证）。*
3. 启动本地开发服务器：
   ```bash
   npm run dev
   ```
4. 浏览器访问 `http://localhost:3000` 即可查看前台，访问 `http://localhost:3000/admin` 进入后台管理。

**默认账号密码与首次使用注意事项**：
- **默认密码**：系统无固定默认密码，完全由 `.env` 文件中的 `ADMIN_PASSWORD` 环境变量决定。
- **首次使用**：请确保你的 PostgreSQL 数据库允许外部连接。系统的 API 会在第一次请求时自动执行 `ensureSchema()` 尝试创建所需的数据表（如 `journal_notes`、`journal_papers` 等）。如果报错，请检查数据库权限。

## 功能特性

1. **静态前端响应式主页**
   - **用户角色**：访客（Public）
   - **业务流程**：访客进入 `http://localhost:3000`，可滑动查看作者的 About、Research、Publications（学术成果）以及 Experience 履历。支持在右上角切换暗黑模式。学术成果部分直接与后台数据库联动，实时展示最新的论文。
2. **安全私密的管理后台**
   - **用户角色**：管理员（Admin）
   - **业务流程**：管理员访问 `/admin`，通过输入环境变量中配置的密码换取加密 Cookie。成功后进入采用左侧垂直吸边导航布局的后台工作台。
3. **今日计划与拖拽管理**
   - **用户角色**：管理员（Admin）
   - **业务流程**：进入后台【今日计划】模块，可添加每日待办事项。通过鼠标拖拽卡片最左侧的句柄，可使用 `@dnd-kit` 实现实时重新排序。
4. **每日打卡与可视化**
   - **用户角色**：管理员（Admin）
   - **业务流程**：进入【每日记录】模块，支持日历与列表双视图切换，可记录体重与私人习惯（如禁欲状态）。附带数据图表展示变化趋势，并支持将记录一键导出为 Markdown 文件。
5. **本地优先的体型追踪**
   - **用户角色**：管理员（Admin）
   - **业务流程**：进入【体型记录】模块，可上传 5-10 张照片。为保护隐私，所有照片直接在浏览器端写入 IndexedDB。系统使用 `html2canvas` 在照片上自动绘制日期水印，并支持两张照片并排对比及 PDF 报告生成导出。
6. **学术论文管理**
   - **用户角色**：管理员（Admin）
   - **业务流程**：进入【论文管理】模块，输入论文元数据（标题、作者、年份、BibTeX）。可直接上传本地 PDF 文件至 `public/papers/`，上传成功后即刻在访客端主页的 Publications 区域提供下载预览链接。
